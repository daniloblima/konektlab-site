<task type="auto">
  <name>Task 1: Media queries para tablet (640px e 768px) e desktop (1024px+)</name>
  <files>css/style.css</files>
  <action>
    Adicionar ao FINAL de css/style.css (APPEND, não substituir) os blocos de media queries.

    A estratégia é mobile-first: o CSS base (sem media query) é para 320px. As media queries adicionam capacidade conforme a tela cresce.

    **Breakpoint 640px — tablets pequenos e landscape mobile:**
    ```css
    /* ===================================================
       RESPONSIVIDADE — 640px (tablet pequeno)
       =================================================== */
    @media (min-width: 640px) {
      h1 {
        font-size: var(--font-size-4xl); /* 3rem → maior no tablet */
      }

      .cards-grid {
        grid-template-columns: repeat(2, 1fr); /* 2 colunas */
      }

      .sobre-content {
        flex-direction: row;
        align-items: flex-start;
        gap: var(--space-8);
      }

      .foto-placeholder {
        width: 200px;
        height: 200px;
        flex-shrink: 0;
      }
    }
    ```

    **Breakpoint 768px — tablets padrão:**
    ```css
    /* ===================================================
       RESPONSIVIDADE — 768px (tablet padrão)
       =================================================== */
    @media (min-width: 768px) {
      section {
        padding-top: var(--space-16);
        padding-bottom: var(--space-16);
      }

      h1 {
        font-size: 3.5rem; /* escala livre entre 3rem e 4rem */
      }

      .subheadline {
        font-size: var(--font-size-2xl);
      }

      .body-text {
        font-size: var(--font-size-xl);
      }

      .situacoes li {
        font-size: var(--font-size-xl);
      }

      .card-servico {
        padding: var(--space-8);
      }

      .card-evidencia {
        padding: var(--space-6);
      }
    }
    ```

    **Breakpoint 1024px — desktop:**
    ```css
    /* ===================================================
       RESPONSIVIDADE — 1024px+ (desktop)
       =================================================== */
    @media (min-width: 1024px) {
      h1 {
        font-size: 4rem; /* 64px no desktop */
      }

      /* Grid de serviços: 3 colunas no desktop */
      .servicos .cards-grid {
        grid-template-columns: repeat(3, 1fr);
      }

      /* Grid de evidências: 3 colunas no desktop */
      .evidencias .cards-grid {
        grid-template-columns: repeat(3, 1fr);
      }

      /* Foto maior no desktop */
      .foto-placeholder {
        width: 240px;
        height: 240px;
      }

      /* Container mais largo para o conteúdo principal */
      .container {
        padding-left: var(--space-6);
        padding-right: var(--space-6);
      }
    }
    ```

    **Regras de responsividade que devem estar garantidas ANTES das media queries (mobile base):**

    Verificar no CSS base (sem media query) que:
    1. `max-width: 100%` está implícito em todo elemento via `box-sizing: border-box`
    2. `.container` tem `padding-left` e `padding-right` para evitar texto colado na borda
    3. `.cta-btn` tem `min-height: 44px` (RESP-04) — já definido no Plan 02
    4. Imagens têm `max-width: 100%` — já definido no Plan 02 via `img { max-width: 100% }`

    Se algum desses itens não estiver no CSS base, adicionar ANTES dos blocos @media.

    **Tipografia responsiva (RESP-03):**
    Adicionar font-size fluido para o h1 usando clamp() como refinamento:
    ```css
    /* Aplicar ANTES dos @media, no bloco h1 base ou sobrescrevendo */
    h1 {
      font-size: clamp(1.75rem, 5vw, 4rem); /* flui de 28px a 64px */
    }
    ```
    Se clamp() foi adicionado, remover os font-size de h1 nas media queries acima (para evitar conflito). Decidir: usar clamp() OU breakpoints explícitos, não os dois.

    Recomendação: usar clamp() para h1 e h2, breakpoints explícitos para grid e padding.

    ```css
    h1 {
      font-size: clamp(1.75rem, 5vw, 4rem);
    }

    h2 {
      font-size: clamp(1.5rem, 4vw, 2.25rem);
    }
    ```
  </action>
  <verify>
    Usar DevTools para simular dispositivos:
    1. iPhone SE (320px): nenhum elemento ultrapassa a largura. Texto legível. CTAs visíveis.
    2. iPad (768px): grid de cards vai para 2 colunas. Foto ao lado da bio.
    3. Desktop (1280px): grid de serviços e evidências em 3 colunas.

    Verificar no CSS: `grep -c "@media" css/style.css` retorna 3+ (pelo menos 3 breakpoints)
  </verify>
  <done>Página funcional em 320px, 768px e 1280px. Grid adapta colunas. Tipografia escala. Nenhum overflow horizontal em mobile.</done>

  <name>Task 2: Revisão final de mobile — overflow, touch targets e espaçamentos</name>
  <files>css/style.css, index.html</files>
  <action>
    Passar revisão sistemática em mobile (simular 320px) e corrigir os problemas mais comuns:

    **Checklist de revisão e correções:**

    1. **Overflow horizontal:** Adicionar ao body:
    ```css
    body {
      overflow-x: hidden; /* Prevenir scroll horizontal em mobile */
    }
    ```

    2. **CTAs com área de toque adequada (RESP-04):**
    Verificar que .cta-btn tem `min-height: 44px`. Se não, adicionar:
    ```css
    .cta-btn {
      min-height: 44px;
      display: inline-flex;
      align-items: center;
      justify-content: center;
    }
    ```

    3. **Listas em mobile:** As situações numeradas e os passos do "Como funciona" devem ter padding-left reduzido em mobile para não cortar texto:
    ```css
    @media (max-width: 480px) {
      .situacoes,
      .passos {
        padding-left: var(--space-3); /* Reduzir de space-4 para space-3 em telas muito pequenas */
      }
    }
    ```

    4. **Header em mobile:** Garantir que o nome "konekt.lab" e o header não transbordem:
    ```css
    header {
      padding: var(--space-3) var(--space-3); /* Ajustar padding em mobile */
    }
    ```

    5. **Cards com altura mínima:** Em mobile com 1 coluna, cards de evidência não devem ficar muito comprimidos. Verificar que o texto está legível. Não precisar de ajuste se o layout já funcionar.

    6. **Verificação final do index.html:** Confirmar que todos os `<section>` têm `<div class="container">` ou `<div class="container container--wide">` internamente. Esta é a causa mais comum de layout quebrado. Se alguma seção estiver sem container, adicionar agora.

    Estrutura esperada de CADA section:
    ```html
    <section id="nome" class="nome">
      <div class="container">  <!-- ou container--wide para grids -->
        <!-- conteúdo -->
      </div>
    </section>
    ```

    7. **Seção hero em mobile:** h1 muito grande pode quebrar em mobile. O clamp() do Task 1 cuida disso, mas verificar visualmente.

    8. **Foto placeholder em mobile:** Deve ser centralizada se ficar sozinha na coluna:
    ```css
    @media (max-width: 639px) {
      .foto-placeholder {
        margin: 0 auto; /* centralizar em mobile */
      }
    }
    ```
